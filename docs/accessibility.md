# Accessibility

The calculator is designed to work across different input methods, screen sizes, and user preferences.

## Current support

- Every calculator control is a native button.
- Controls have accessible labels, including **Change sign** and **Percent**.
- Results are announced through a polite live region.
- Keyboard users receive a visible focus indicator.
- Touch targets remain comfortably sized on narrow screens.
- Meaning is not communicated by color alone.
- Reduced-motion preferences disable button transitions.
- The interface uses responsive type and layouts instead of fixed desktop dimensions.

## Documentation accessibility

The documentation provides:

- keyboard-accessible navigation and search
- light and dark themes
- semantic headings and tables
- descriptive link text
- short sections written in plain language

## Reporting a problem

If a control, page, or explanation creates an accessibility barrier, [open a GitHub issue](https://github.com/TechWriterP/calculator/issues/new). Include:

1. what you were trying to do
2. the device, browser, and assistive technology used
3. what happened and what you expected

Do not include private or sensitive information.
